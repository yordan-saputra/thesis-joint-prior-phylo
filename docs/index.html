Thesis Draft 1

Joint prior distributions for hierarchical phylogenetic models

Author

Yordan Saputra

Problem Description

Phylogenetic trees are used to represent the evolutionary history between a set of species. The implied dependencies between species can be expressed statistically via phylogenetic models. However, these models are often hard to estimate and may require strong prior knowledge, which we express in Bayesian statistics via prior distributions. The goal of this thesis is to implement and evaluate joint prior distributions for the variance parameters of phylogenetic models. We have concrete ideas for such priors, which we will discuss with you upon starting your thesis.

Project Structure

  • Literature search on priors for phylogenetic models
  • Implement the new and existing priors in Stan
  • Run simulation studies
  • Analyse and discuss results

load_or_fit_brms <- function (data, model_name, formula, family, data2, prior, replace = FALSE) {
  modelpath <- file.path(getwd(), 'PhyloModel', paste0(model_name, '.rds'))
  
  if (file.exists(modelpath) && !replace) {
    model <- readRDS(modelpath)
  } else {
    model <- brm(
      formula, data = data, family = family, 
      data2 = data2, prior = prior, 
      refresh = 0, silent = 2, 
      control = list(adapt_delta = 0.95)
    )
    saveRDS(model, modelpath)
  }
  
  return(model)
}

load_or_fit_stan <- function (data, model_name, stan_model, replace = FALSE) {
  modelpath <- file.path(getwd(), 'PhyloModel', paste0(model_name, '.rds'))
  stanpath <- file.path(getwd(), 'PhyloModel', paste0(model_name, '.stan'))
  
  if (file.exists(modelpath) && !replace) {
    model <- readRDS(modelpath)
  } else {
    # writeLines(stan_model@model_code[1], stanpath)
    data_list <- list(
      N = nrow(data), y = data$phen, 
      x1 = data$spec_mean_cf, x2 = data$within_spec_cf,
      J = length(unique(data$species)),
      group = as.numeric(factor(data$species)),
      A_phylo = A
    )
    model <- sampling(
      stan_model, data = data_list, 
      chains = 4, warmup = 1000, iter = 2000,
      cores = 4, seed = 67,
      control = list(adapt_delta = 0.95, max_treedepth = 12)
    )
    saveRDS(model, modelpath)
  }
  
  return(model)
}

  real beta_w; // Slope for within_spec_cf
  
  // Variance components
  real<lower=0> sigma_phy; // SD for phylogenetic effect
  real<lower=0> sigma_sp; // SD of species random effects
  
  // Non-centered parameterization
  vector[J] phy_eff_z;// latent variable for phylogenetic
  vector[J] sp_eff_z; // latent variable for species
  
  // Residual error
  real<lower=0> sigma;
}

transformed parameters {
  vector[J] phy_eff = sigma_phy * (cholesky_decompose(A_phylo) * phy_eff_z); // phylogenetic effect (a)
  vector[J] sp_eff = sigma_sp * sp_eff_z; // species effect (s)
}

model {
  // Priors
  mu ~ normal(0, 50);
  beta_b ~ normal(0, 10);
  beta_w ~ normal(0, 10);
  
  sigma_phy ~ student_t(3, 0, 20);
  sigma_sp ~ student_t(3, 0, 20);

Phylogenetic Generalized Linear Mixed Model (PGLMM) is the GLMM version for phylogenetic datasets. Although, in this case, it’s more appropriate to call it a phylogenetic (linear) mixed model (PMM) since the output being modeled is Gaussian. Maybe we can explore more of this later.

Note: The notation used here is \(\mathcal{N}(\mu,\sigma)\), where \(\sigma\) is the standard deviation, not variance.

\[\begin{gather} \boldsymbol{y} = \mu + \beta_B\ \bar{\boldsymbol{x}} + \beta_W\ (\boldsymbol{x} - \bar{\boldsymbol{x}}) + \boldsymbol{a} + \boldsymbol{s} + \boldsymbol{e} \\ \boldsymbol{a} \sim \mathcal{N}\left(\boldsymbol{0}, \sigma_P\boldsymbol{L}\right),\quad \boldsymbol{\Sigma} = \boldsymbol{L}\boldsymbol{L}^\top \\ \boldsymbol{s} \sim \mathcal{N}\left(\boldsymbol{0}, \sigma_S\boldsymbol{I}\right),\quad \boldsymbol{e} \sim \mathcal{N}\left(\boldsymbol{0}, \sigma_R\boldsymbol{I}\right) \\ \boldsymbol{y} \sim \mathcal{N}\left(\mu + \beta \boldsymbol{x}, \boldsymbol{\Lambda} \right),\quad \sigma_p^2\boldsymbol{\Sigma} + \sigma_R^2\boldsymbol{I} = \boldsymbol{\Lambda}\boldsymbol{\Lambda}^\top \\ \end{gather}\]

For clarification, consider a dataset of 15 different species where each species is measured 5 times (of course, the number of measurements for each species can differ; the example below is just for easier illustration):

\[\begin{gather} \begin{bmatrix} y_{1,1} \\ y_{1,2} \\ \vdots \\ y_{i,j} \\ \vdots \\ y_{15,4} \\ y_{15,5} \\ \end{bmatrix} = \mu + \beta_B \begin{bmatrix} x_{1} \\ x_{1} \\ \vdots \\ x_{i} \\ \vdots \\ x_{15} \\ x_{15} \\ \end{bmatrix} + \beta_W \begin{bmatrix} x_{1,1} - x_1 \\ x_{1,2} - x_1 \\ \vdots \\ x_{i,j} - x_i \\ \vdots \\ x_{15,4} - x_{15} \\ x_{15,5} - x_{15} \\ \end{bmatrix} + \begin{bmatrix} a_{1} \\ a_{1} \\ \vdots \\ a_{i} \\ \vdots \\ a_{15} \\ a_{15} \\ \end{bmatrix} + \begin{bmatrix} s_{1} \\ s_{1} \\ \vdots \\ s_{i} \\ \vdots \\ s_{15} \\ s_{15} \\ \end{bmatrix} + \begin{bmatrix} e_{1,1} \\ e_{1,2} \\ \vdots \\ e_{i,j} \\ \vdots \\ e_{15,4} \\ e_{15,5} \\ \end{bmatrix} \end{gather}\]

where:

\[\begin{gather} \boldsymbol{y} \text{ is the phenotype data of several species, for example, the body size of the species} \\ y_{i,j} \text{ is the phenotype data (body size) of individual } j \text{ belonging to species } i \\ \\ \boldsymbol{x} \text{ is the co-factor of interest, for example, the temperature of the environment} \\ x_{i,j} \text{ is the co-factor (environment temperature) of individual } j \text{ belonging to species } i \\ x_i = \frac{1}{J_i}\sum_{i=1}^{J_i} x_{i,j} \text{ is the mean co-factor of interest of species } i \\ J_i \text{ is the number of individuals in species } i \\ \\ \boldsymbol{a} \text{ is the phylogenetic random effect} \\ a_i \text{ is the phylogenetic random effect of species } i \\ \\ \boldsymbol{s} \text{ is the multiple measurement effect / non-phylogenetic species-specific effect} \\ s_i \text{ is the multiple measurement effect of species } i \\ \\ \boldsymbol{e} \text{ is the residual error} \\ e_{i,j} \text{ is the residual error of individual } j \text{ belonging to species } i \\ \\ \mu, \beta_B, \beta_W \text{ are the (1) intercept, (2) slope between species, and (3) slope within species for co-factor } \boldsymbol{x} \\ \sigma_P, \sigma_S, \sigma_R \text{ are the standard deviation parameters for } \boldsymbol{a}, \boldsymbol{s}, \boldsymbol{e} \end{gather}\]

Besag York Mollié (BYM) model includes both an ICAR component for spatial smoothing and an ordinary random-effects component for non-spatial heterogeneity.

\[\begin{gather} Y_i\ |\ \psi_i \sim Poisson\left(E_i e^{\psi_i}\right) \\ \boldsymbol{\psi} = \boldsymbol{x}\beta + \boldsymbol{\theta} + \boldsymbol{\phi} \end{gather}\]

The problem with this model is that fitting a model which includes both an ordinary random effects component \(\boldsymbol{\theta}\) and a spatial ICAR component \(\boldsymbol{\phi}\) is difficult because either component can account for most or all of the individual-level variance. To handle this, the BYM2 model has been proposed, which places a single precision (scale) parameter on the combined components and a mixing parameter for the amount of spatial versus non-spatial variation.

\[\begin{gather} \boldsymbol{\theta} + \boldsymbol{\phi} = \sigma\left(\sqrt{1-\rho}\ \boldsymbol{\theta}^* + \sqrt{\rho}\ \boldsymbol{\phi}^* \right) \\ \text{where Var}(\theta_i) \approx \text{Var}(\phi_i) \approx 1 \end{gather}\]
